# LeetCode SOP

## 每日流程

1. 选择当天主题，例如数组、哈希、双指针、树、图、DP 或贪心。
2. 先做 1 道基础/中等题建立手感，再做 1 道同类变体。
3. 每题先写暴力解和复杂度，再推导优化解。
4. 用 5 分钟整理模板、边界条件和失败原因。
5. 将关键题型沉淀到当天学习日志或专题笔记。

## 评估标准

- 能否在不看答案的情况下讲清楚状态定义、转移或不变量。
- 能否写出时间复杂度和空间复杂度，并说明优化来源。
- 是否记录了边界条件，例如空输入、重复元素、越界、负数、环。
- 是否能把同类题归纳为一个可复用模板，而不是只记单题答案。

## 主题顺序

```text
数组/字符串 -> 哈希/双指针 -> 链表/栈/队列 -> 树 -> 回溯 -> 图 -> DP -> 贪心 -> 高频综合
```
